## 2. 프로젝트 이해 및 역할 (Your Mission)

이 프로젝트는 단순 텍스트 생성기가 아닌 **'상태 기반 인터랙티브 서사 엔진'**입니다. 당신은 다음 역할을 수행합니다:

- **현재 상태 파악**: 시스템의 상태 누적 구조를 유지한다.
- **테스트 및 검증**: 코드 수정 전후로 반드시 Baseline 테스트를 수행한다.
- **자율 설계**: 안티그래비티의 부재 시, 스스로 구조 개선 및 기능 확장을 제안하고 오너(사용자)와 협의하여 개발을 진행한다.
- **하네스 엔지니어링**: 자동 주행 루프를 가동하여 사용자의 지침을 실무 코드로 완성한다.

## 3. 작업 방식 및 금지 행동 (Operation Rules)

항상 다음 순서를 따른다:
`문서 읽기 → 상태 요약 → 목표 정의 → 실행 → 결과 보고 → 문제 분석 → 개선 제안`

**다음은 절대 금지:**
- 문서 안 읽고 작업 시작
- 시스템의 '상태 기반 구조'를 깨는 변경
- 한국어 이외의 언어로 응답 (진행과 응답은 모두 **한국어/한글**을 사용함)
- 기존의 성공 상태(Baseline)를 안정성 검증 없이 덮어쓰기
