# Story Engine Rule
# 안티그래비티가 코드/구조 다룰 때 지켜야 할 규칙

## 0. 목적

이 규칙은 코드 작업 시
프로젝트 구조가 무너지는 것을 방지하기 위한 것이다.

---

## 1. 절대 원칙

이 프로젝트는:

👉 상태 기반 서사 엔진이다

이 원칙을 코드 레벨에서도 반드시 유지해야 한다.

---

## 2. 핵심 구조 보호

다음 요소는 절대 훼손 금지:

- Scene
- Choice
- State
- Emotion
- Relation
- Memory
- Goal
- Route
- Ending

---

## 3. 코드 수정 규칙

코드 수정 시 반드시:

1. 기존 로직 이해
2. 영향 범위 파악
3. 테스트 기준 확인
4. 수정 진행
5. 테스트 실행

---

## 4. 금지 사항

- 구조 모르는 상태에서 수정 ❌
- 전체 갈아엎기 ❌
- 테스트 없이 변경 ❌
- 기존 기능 파괴 ❌

---

## 5. 테스트 기준

수정 후 반드시:

- test_run.py 실행
- 3루트 테스트 확인
- 회귀 체크 확인

---

## 6. 설계 참여 기준

Antigravity는 설계도 참여 가능하다.

단,

- 구조 유지
- 상태 기반 유지
- 문서 기준 준수

이 3개는 반드시 지켜야 한다.

---

## 7. 한 줄 핵심

👉 “코드는 바꿀 수 있어도, 구조는 못 바꾼다”
