## 0. 이 문서의 목적

이 문서는 현재 프로젝트가 어디까지 왔는지,
지금 무엇이 성공 상태인지,
다음에 무엇을 해야 하는지,
ChatGPT가 없더라도 오너와 Antigravity가 바로 이어서 작업할 수 있도록 하기 위한 문서다.

즉, 이 문서는 “현재 상태판”이다.

---

## 1. 프로젝트 현재 정의

현재 프로젝트는
**선택 기반 상태 누적형 인터랙티브 창작 엔진**
또는
**카드 기반 누적형 서사 생성 시스템**
을 만드는 프로젝트다.

이 시스템은 단순한 글 생성기가 아니라,
장면 / 선택 / 감정 / 관계 / 목표 / 기억 / 루트 / 엔딩이 함께 움직이는
상태 기반 서사 시스템이다.

---

## 2. 현재까지 구현된 상태

현재까지 확인된 핵심 상태는 다음과 같다.

### 2-1. 엔진 구조
- 카드 기반 구조 구현
- 감정 / 관계 / 목표 / 기억 / 루트 구조 구현
- 장면 생성 구조 구현
- 추천 생성 구조 구현
- 루트 전이 구조 구현
- 엔딩 판정 구조 구현

### 2-2. 테스트 구조
- `test_run.py` 존재
- 3루트 테스트 존재
  - 고백 루트
  - 회복 루트
  - 파국 루트
- 멀티 시드 테스트 존재
- 회귀 체크 존재

### 2-3. 출력 구조
- 장면 출력
- 추천 출력
- 상태창 출력
- 엔딩 카드 출력
- 최종 요약 리포트 출력
- Markdown export
- JSON export

## 4. 현재 성공 기준 (Baseline)

다음 항목을 만족하면 현재 프로젝트는 정상 상태로 본다.

1. `python3 test_run.py` 실행 가능
2. 고백 / 회복 / 파국 3루트 테스트 정상 실행
3. 회귀 체크 PASS
4. Markdown / JSON 산출 생성
5. 전체 장면 흐름이 중간에 끊기지 않음

이 성공 기준은 가장 중요하다.
새 기능보다 먼저 이 기준을 유지해야 한다.

## 8. 현재 협업 원칙

이 프로젝트는 특정 AI 하나에 의존하지 않는다.

협업 구조는 다음과 같다.

### ChatGPT
- 고급 설계
- 구조 해석
- 철학 정리
- 문제 원인 분석
- 문서화 지원

### Antigravity
- 현재 상태 파악
- 문서 읽기
- 실행
- 검증
- 로그 분석
- 설계 참여
- ChatGPT 부재 시 오너와 함께 진행

### 오너
- 최종 판단
- 방향 결정
- 우선순위 선택
- 실제 작업 진행 승인

---

## 9. ChatGPT 부재 시 현재 운영 원칙

ChatGPT가 없는 경우에도 프로젝트는 멈추지 않아야 한다.

그 경우:
- 오너 + Antigravity가 현재 문서를 기준으로 진행한다
- 현재 성공 기준을 먼저 확인한다
- 테스트를 먼저 수행한다
- 필요하면 보수적으로 설계/개선한다
- 구조를 깨지 않는 선에서 작업한다

즉,
현재 시스템은 “ChatGPT가 없으면 멈추는 구조”가 아니라
“문서와 협업으로 이어지는 구조”를 목표로 한다.

## 5. 절대 착각 금지
현재 시스템은 "완성된 엔진"이 아니라
"동작하는 레거시 코드 조각"에 가깝다.

따라서 안정성을 검증하지 않은 상태에서의 기능 확장은
전체 시스템 붕괴로 직결될 수 있다.
